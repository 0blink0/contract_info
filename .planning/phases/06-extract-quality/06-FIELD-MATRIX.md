# Phase 6：字段可抽取性与可空性调研

**调研日期：** 2026-05-26  
**样本：** `example/石云中证1000…(1).docx`、`example/石云福禄1000…(1).docx`；黄金表 `产品要素 - 副本(1).xlsx`、`产品运营费率导入模板.xlsx`、`产品申赎费率导入模板.xlsx`  
**用途：** 约束 Phase 6 黄金测试、Critical 字段清单、空值是否合理；**不作为线上运行时校验真值**。

---

## 1. 核心结论

1. **图中 12 类业务要素在基金合同正文中均有依据**，不存在「合同完全没有」的项。  
2. **并非黄金表每一格都应从合同抽出**：黄金表混有 **系统编码、成立后运营数据、与合同不一致的当事人**。  
3. **空值分四类**（见 §2）；测试断言必须区分 **合同真值** vs **黄金表真值**。  
4. **交付阶段**：12 项中 **10 项** 已在抽取链路或 4 类 Excel 中覆盖；**认申赎费 Excel** → Phase 7；**业绩报酬 / 开放日完整管理** → Phase 8。

## 2. 空值四分类（可空性规则）

| 类别 | 含义 | 抽取结果为空 | 与黄金表不一致 |
|------|------|--------------|----------------|
| **A. 非合同源** | 系统/运营录入，合同本无 | **合理** | **允许**（黄金可有值） |
| **B. 合同可选** | 合同可能有，模板/运营常不填 | **合理** | **允许**（黄金常空） |
| **C. 合同应有** | P1/图中要素，合同有表述 | **需告警/修复** | 与 **合同** 对齐，不以黄金为准 |
| **D. 延后交付** | 合同可抽，本里程碑未导出 | 导出为空 **合理** | Phase 7/8 再验 |

### A. 非合同源（空合理，勿与黄金比）

| 字段/数据 | 说明 |
|-----------|------|
| `基金代码`（HXZC38、FLZB38、BPL38A…） | 内部/协会编码，运营系统分配 |
| `运行状态` | 运营状态（如「运行中」），非合同条款 |
| `销售经理`、`产品经理`、`投资机构`、`补充协议`、`特殊的产品类型` | `MANUAL_ONLY_PRODUCT`，不自动抽 |
| `成立日期`、`备案日期` | 合同签署时常无；黄金填了日期 = **成立后补录**，合同抽取为空合理 |

### B. 合同可选（主表空常见）

黄金表每基金约 **填 37 列、空 28 列**；下列在黄金中常空，合同可能有，**不强制 Phase 6 填满**：

基金简称、到期/清算日期、投资顾问、外包机构（主表）、产品分类、母基金代码、默认分红方式、合同变更方式、量化相关、封闭细节、转换/赎回方式、销售机构信息、银行账户信息等。

**特例：** 石云进取合同有 **外包机构、锁定期、投资经理**，黄金主表却空 —— 属 **运营填表习惯或填在子表**，抽取有值 **不算错**。

### C. 合同应有（Phase 6 Critical，对齐合同）

见 §3「图中 12 项」与 §4「石云样本合同真值」。

### D. 延后交付

| 内容 | 阶段 |
|------|------|
| 认申赎费（认购/申购/赎回） | Phase 7 → 申赎费率 xlsx |
| 业绩报酬结构化 | Phase 8 → 路径 B JSON |
| 开放日管理（临时开放、仅赎回等） | Phase 8 → 路径 B JSON |

## 6. Phase 6 Critical 字段清单（建议）

**合同对齐（必验，石云进取 + 福禄各一套 expected）：**

- 基金全称、基金简称（若合同有）  
- 管理人、托管人  
- 风险等级  
- 投资经理  
- 投资目标、投资范围、投资限制、投资策略（`@pytest.mark.llm`）  
- 预警线、止损线  
- 锁定期（有条款时）或子表锁定期行  
- 开放日规则（摘要）  
- 运营费率：管理费、托管费、销售服务费、基金服务费 及 **费率（%/年）**（按份额类行数与黄金结构比）

**允许空（不测失败）：**

- `MANUAL_ONLY_PRODUCT` 六列、`基金代码`、`成立日期`、`备案日期`（合同无）  
- 黄金有而合同无的运维字段  

**禁止用黄金判定当事人：**

- 管理人、托管人、外包机构 — 以合同为准
