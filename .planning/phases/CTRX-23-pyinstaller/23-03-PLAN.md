<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Electron main process → backend child process env | backendChildEnv() provides the env; explicit keys after ...process.env spread override system values |
| installer → electron/resources/models/ | extraResources filter controls which files are bundled; missing entry = absent model at runtime |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-05 | Spoofing | CTRX_MODELS_DIR system env var override | mitigate | Explicit key after ...process.env spread ensures injected value always wins (D-12). Verified by JavaScript object spread semantics: `{ ...obj, KEY: val }` — `val` overrides `obj.KEY`. |
| T-23-06 | Tampering | D-08 model-dir guard bypass | accept | Guard is defensive (fast-fail on missing directory). An attacker who can remove the model directory from the install location can also bypass the guard; no additional mitigation needed for a local desktop app. |
| T-23-07 | Denial of Service | extraResources models/** bundling increases installer size ~2.5 GB | accept | Size impact is documented and expected (D-07: model must be pre-bundled). No user surprise; installer size communicated in release notes. |
| T-23-SC | Tampering | npm/package.json modifications | accept | package.json is a trusted project file; no supply chain concern for filter string addition. |
</threat_model>
