useKbEntry 函数签名：
```typescript
export function useKbEntry(
  jobId: Ref<string | null>,
  jobFilename: Ref<string | null>,
  crmRows: Ref<CrmHandoffItem[]>,
)
```

内部 refs：
- kbRows: ref<KbRow[]>([])
- submitting: ref(false)
- kbUnavailable: ref(false)

buildRows() 函数（同步，供 watch 调用）：
  - kbRows.value = KB_FIELDS.map((fieldName) => { const row = crmRows.value.find((r) => r.crm_field === fieldName); return { field_name: fieldName, field_value: row?.suggested_value ?? '', snippet: row?.snippet ?? '', selected: false } })

submit() 异步函数：
  - const selected = kbRows.value.filter((r) => r.selected)
  - if (!selected.length || !jobId.value) return
  - submitting.value = true
  - try:
    - const res = await postKbEntries({ entries: selected.map((r) => ({ field_name: r.field_name, field_value: r.field_value, snippet: r.snippet, source_job_id: jobId.value!, source_filename: jobFilename.value ?? '' })) })
    - ElMessage.success(`已存入 ${res.count} 条`)（文案合同精确字符串）
  - catch (e):
    - 503 检测：if (e instanceof Error && e.message.includes('503')) { kbUnavailable.value = true; ElMessage.error('知识库不可用，请检查模型是否已加载') }（文案合同专用字符串）
    - 其他错误：ElMessage.error(e instanceof Error ? e.message : '存入失败')
  - finally: submitting.value = false

返回：{ kbRows, submitting, kbUnavailable, buildRows, submit }

注意：
- import { ref, type Ref } from 'vue'（不用 reactive）
- import { ElMessage } from 'element-plus'
- import type { CrmHandoffItem } from '@/api/types'
- import { postKbEntries } from '@/api/kb'
- 不在 useKbEntry 内部调用 buildRows()（由 PathBDetail.vue 的 watch(data, buildRows) 驱动）
