<threat_model>
## 信任边界

| 边界 | 说明 |
|------|------|
| 客户端 → POST /kb/entries | 未受信任输入：entries 数组可能含过长列表（DoS）或 XSS 字符串 |
| 客户端 → DELETE /kb/entries/{id} | 未受信任输入：entry_id 路径参数可能含注入字符 |

## STRIDE 威胁清单

| 威胁 ID | 类别 | 组件 | 处置 | 缓解措施 |
|---------|------|------|------|---------|
| T-20-02-01 | Tampering | DELETE /kb/entries/{id} — LanceDB where 子句拼接 | mitigate | delete_kb_entry 调用前执行 `uuid.UUID(entry_id)` 校验；非 UUID 格式返回 422，阻止注入 `;DROP TABLE` 类攻击 |
| T-20-02-02 | DoS | POST /kb/entries — entries 数组过长 | mitigate | KbEntriesRequest.entries 使用 `Field(..., max_length=20)`；单批次最多 20 条，防止超大 embedding 请求阻塞线程池 |
| T-20-02-03 | Elevation | routes/kb.py router 未鉴权 | mitigate | router 声明 `dependencies=[Depends(verify_api_key)]`，与现有 jobs.py 一致；所有 /kb 端点均需 X-API-Key 头 |
| T-20-02-04 | DoS | asyncio.to_thread(init_kb_service) — bge-m3 加载时间过长 | accept | bge-m3 首次加载 10-30s（CPU），用 asyncio.to_thread 不阻塞事件循环；Electron IPC 有独立 Python ready 超时机制；RESEARCH.md Open Question #1 已记录 |
</threat_model>
