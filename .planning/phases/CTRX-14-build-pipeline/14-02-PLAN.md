<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build script parameter → subcommand invocation | -Version / --version value is passed into child process arguments and into a JSON string |
| build script → npm/npx invocation | npm run build and npx commands execute code from node_modules installed in Wave 0 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-05 | Tampering | -Version / --version parameter injection | mitigate | Both scripts validate the version value against regex ^\d+\.\d+\.\d+$ before passing it to any subcommand; non-matching input causes fail-fast exit with error message (per RESEARCH.md Security Domain: "Build scripts validate version param format") |
| T-14-06 | Tampering | --extraMetadata JSON string construction | mitigate | Version is validated as ^[0-9]+\.[0-9]+\.[0-9]+$ (digits and dots only) before string interpolation into JSON; no shell metacharacters possible in a pure semver string |
| T-14-07 | Denial of Service | Step N failure silently absorbed | mitigate | $ErrorActionPreference = "Stop" (PS) and set -euo pipefail (Bash) ensure any non-zero exit code from npm/npx/pyinstaller terminates the entire script immediately |
| T-14-08 | Repudiation | NSIS installer triggers SmartScreen | accept | Unsigned NSIS installer on unsigned publisher triggers SmartScreen reputation warning; this is expected per RESEARCH.md Pitfall 3; code signing is out of scope per REQUIREMENTS.md; users instructed to click "More info -> Run anyway" for internal distribution |
| T-14-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installs in this plan; build scripts invoke npx which runs packages already installed in Wave 0 (node_modules); slopcheck passed for all packages in 14-01 |
</threat_model>
