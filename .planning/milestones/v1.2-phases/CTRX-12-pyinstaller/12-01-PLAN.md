<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build scripts -> packaged runtime | 构建产物把本地依赖固化到可分发目录，若漏包会在无 Python 环境崩溃 |
| packaged backend -> local filesystem | 后端启动会读写 `CTRX_DATA_DIR`，需确保不写入安装资源目录 |
| smoke harness -> release decision | 烟测结果直接决定是否可发布，需防止“假阳性通过” |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | T | `ctrx_backend.spec` hiddenimports | mitigate | 维护可审计 hiddenimports 清单与变更日志，构建失败即阻断 |
| T-12-02 | D | `desktop_main.py` 资源加载 | mitigate | 启动前检查 `alembic/dicts/templates`，缺失 fail-fast（per D-04） |
| T-12-03 | I | `electron/resources/` 目录 | mitigate | 日志/临时文件外置到 `CTRX_DATA_DIR`，避免安装目录写入（per D-11） |
| T-12-04 | R | smoke 结果记录 | mitigate | 统一脚本生成 checklist + 日志 + 截图路径，失败返回非零退出 |
| T-12-05 | E | 数据库连接配置 | mitigate | 启动链强制 sqlite URL + cache_clear，测试断言不连接 PostgreSQL |
</threat_model>
