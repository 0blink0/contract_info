## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| installer → end user machine | NSIS installer runs on clean VM; unsigned binary triggers SmartScreen |
| AppImage/deb → Ubuntu 22.04 runtime | glibc version of build host must be <= glibc of target machine |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-09 | Repudiation | NSIS installer SmartScreen warning | accept | --onedir PyInstaller (no self-extracting .exe) already mitigates AV false-positive trigger; SmartScreen reputation check for unsigned binary is expected; documented as "More info -> Run anyway" for internal distribution; code signing explicitly out of scope in REQUIREMENTS.md |
| T-14-10 | Tampering | extraResources path mismatch at runtime | mitigate | Task 1 Step E verifies resources/electron/resources/.backend-manifest.json exists inside installed app; if missing, backendEntrypoint() falls back to hardcoded path and the application fails to start — this is caught by human verification in Task 2 |
| T-14-11 | Elevation of Privilege | NSIS perMachine: false setting | accept | Per-user install (perMachine: false) avoids UAC elevation prompt, reducing Defender scrutiny and requiring no admin rights; correct for internal distribution |
| T-14-12 | Denial of Service | Linux AppImage built on wrong glibc | mitigate | Task 2 human verification explicitly requires Ubuntu 22.04 (NOT 24.x); if AppImage crashes with glibc error, rebuild on Ubuntu 22.04 per RESEARCH.md Pitfall 4 |
| T-14-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan; only running existing installed tools (npx tsc, npx electron-builder); verified in 14-01 |
</threat_model>
