From RESEARCH.md DB-04 section (target desktop_main.py):
  def run_migrations(db_url: str, alembic_dir: Path) -> None:
      from alembic.config import Config
      from alembic import command
      alembic_cfg = Config()
      alembic_cfg.set_main_option("script_location", str(alembic_dir))
      alembic_cfg.set_main_option("sqlalchemy.url", db_url)
      command.upgrade(alembic_cfg, "head")

  def main() -> None:
      bundle_base = _get_bundle_base()
      data_path = Path(os.environ.get("CTRX_DATA_DIR") or Path.home() / ".ctrx")
      data_path.mkdir(parents=True, exist_ok=True)
      db_url = f"sqlite:///{data_path / 'ctrx.db'}"
      os.environ["DATABASE_URL"] = db_url
      os.environ["CTRX_DATA_DIR"] = str(data_path)
      os.environ.setdefault("CTRX_PORT", "8765")
      from backend.app.config import get_settings
      get_settings.cache_clear()
      alembic_dir = bundle_base / "alembic"
      run_migrations(db_url, alembic_dir)
      import uvicorn
      port = int(os.environ.get("CTRX_PORT", "8765"))
      uvicorn.run("backend.app.main:app", host="127.0.0.1", port=port)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| desktop_main → backend imports | All env var setup happens before any backend module is imported; lru_cache cleared to prevent stale Settings |
| WAL pragma → connection pool | Event listener fires on every new connection; busy_timeout is per-connection (must be in event listener, not just at startup) |
| alembic upgrade → SQLite file | Alembic creates ctrx.db in CTRX_DATA_DIR; directory must be writable by the process user |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-09 | Denial of Service | busy_timeout=5000 — concurrent writers wait up to 5 seconds before raising OperationalError | accept | 5-second timeout is sufficient for desktop single-user app; unlikely to have 2+ simultaneous writes |
| T-11-10 | Tampering | desktop_main.py sets DATABASE_URL env var — could be accidentally overridden by user env | accept | os.environ.setdefault is used for CTRX_PORT (respects existing); DATABASE_URL is set explicitly (intentional override for desktop mode); in dev mode desktop_main.py is not used |
| T-11-11 | Elevation of Privilege | run_migrations creates tables in CTRX_DATA_DIR — must be writable | mitigate | data_path.mkdir(parents=True, exist_ok=True) in main() ensures directory exists; if path is read-only, OperationalError surfaces immediately with clear message; no silent data loss |
| T-11-12 | Information Disclosure | ctrx.db stored in user home/.ctrx by default | accept | Default path ~/.ctrx is user-controlled, not world-readable on any OS with standard umask; no additional encryption needed for desktop internal tool |
| T-11-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this phase |
</threat_model>
