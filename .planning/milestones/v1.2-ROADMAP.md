# Milestone v1.2: 桌面化交付

**Status:** ✅ SHIPPED 2026-05-29
**Phases:** 11–14
**Total Plans:** 11

## Overview

将 CTRX 从 Docker-only 部署转型为 Electron 桌面应用，双击安装包即用。Phase 11 完成 SQLite 迁移（替换 PostgreSQL 方言类型、路径隔离、WAL 模式、programmatic Alembic）；Phase 12 完成 PyInstaller 打包（单 spec、hiddenimports 审计、Windows 主链路烟测）；Phase 13 完成 Electron 壳（生命周期状态机、3 通道 IPC、首启向导、Settings 重启回滚）；Phase 14 完成构建流水线（electron-builder 26.x NSIS + AppImage + deb，产出 CTRX-Setup-1.2.0.exe 133MB）。

**Goal**: 一条命令产出可分发的 Windows NSIS 安装包（.exe）、Linux AppImage 和 .deb，构建步骤有文档脚本
**Depends on**: Phase 13
**Requirements**: BUILD-01, BUILD-02, BUILD-03

Plans (Wave structure):

- [x] 14-02-PLAN.md — Wave 1: scripts/build.ps1 + scripts/build.sh 4-step dispatchers with fail-fast + version injection (BUILD-02)
- [x] 14-03-PLAN.md — Wave 2: tsc dry-run + Windows build smoke test + human acceptance gate (BUILD-01, BUILD-02, BUILD-03)

**Details:**

Key deliverables:
- `package.json`: electron-builder 26.8.1 inline config (appId: com.ctrx.app, NSIS+AppImage+deb, extraResources→electron/resources, signAndEditExecutable: false)
- `tsconfig.electron.json`: NodeNext module/moduleResolution, allowImportingTsExtensions + rewriteRelativeImportExtensions
- `scripts/build.ps1`: Windows 4-step dispatcher (PyInstaller→Vite→tsc→electron-builder NSIS), semver validation, -c.extraMetadata.version injection
- `scripts/build.sh`: Linux 4-step dispatcher (same, AppImage+deb), Ubuntu 22.04 / glibc 2.35 note
- `dist/CTRX-Setup-1.2.0.exe`: 133MB Windows NSIS installer (electron-builder 26.8.1)

Key fixes discovered:
- TypeScript 6 TS5097: requires allowImportingTsExtensions + rewriteRelativeImportExtensions pair for .ts import paths with NodeNext emit
- electron-builder 26.x: version injection is `-c.extraMetadata.version=X` not `--extraMetadata JSON blob`
- winCodeSign symlink failure: fixed via `signAndEditExecutable: false` (code signing is out of scope per REQUIREMENTS.md)

## Milestone Summary

**Key Accomplishments:**
1. SQLite 完整迁移：PostgreSQL 方言类型全替换，WAL 模式开启，CTRX_DATA_DIR 路径隔离
2. PyInstaller 单 spec 打包：hiddenimports 审计基线 + CI 门禁 + Windows 主链路烟测通过
3. Electron 主进程状态机：子进程生命周期管理、健康轮询、崩溃自动重试（最多 3 次）、SIGTERM→SIGKILL 优雅退出
4. 首启向导 + Settings：三步引导向导（向导未完成封闭主界面）+ Settings 页面保存后重启回滚
5. 构建流水线：electron-builder 26.x NSIS + AppImage + deb 一键构建，产出 CTRX-Setup-1.2.0.exe（133MB）
6. 关键修复：TypeScript 6 NodeNext 双标志、electron-builder 26.x 版本注入语法、signAndEditExecutable=false

**Key Decisions:**
- `--onedir` PyInstaller（非 `--onefile`）— 规避 AV 误报风险
- `electron-store` pin v10（CommonJS require() 兼容）
- SQLite 替换 PostgreSQL；Docker 路径保留不变
- TypeScript 6 NodeNext emit 模式：allowImportingTsExtensions + rewriteRelativeImportExtensions 必须同时存在
- electron-builder 26.x 版本注入语法：`-c.extraMetadata.version=X`（不是 `--extraMetadata JSON blob`）
- `signAndEditExecutable: false`：跳过 winCodeSign 符号链接提取（代码签名超出 v1.2 范围）

**Issues Deferred:**
- PKG-03 Linux clean-VM 烟测（用户决定暂时跳过）
- PATHB-EX-01/02 路径 B 枚举映射增强 → v1.3
- BATCH-01/02/03 批量上传队列 → v2

**Technical Debt:**
- Linux clean-VM 验证缺失：需在 Linux 主机或 CI runner 上跑 `scripts/package_backend.sh` + `scripts/build.sh`
- ESLint v10 flat config 缺失（lint 报 "requires eslint.config.*"）
- Phase 13 preload.cjs 未在 dist/ 落位（CR-01 open item）
