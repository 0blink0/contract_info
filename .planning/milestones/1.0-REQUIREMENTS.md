# Requirements: 合同要素抽取（CTRX） — v1.0

**Defined:** 2026-05-25  
**Shipped:** 2026-05-26  
**Core Value:** 上传 docx → 得到可导入的 Excel，减少手抄模板

## v1 Requirements (final status)

### Document Ingest

- [x] **DOC-01**: 用户可上传单个 `.docx` 合同文件
- [x] **DOC-02**: 系统将 docx 解析为结构化 Document JSON（段落、表格、章节锚点）
- [x] **DOC-03**: 每个上传文件在 PostgreSQL 有唯一记录（id、文件名、状态、时间戳）

### Extraction

- [x] **EXT-01**: 按 `FIELD_SPEC.md` P1 抽取产品要素字段（规则 + LLM）；v1.1 扩展更广字段
- [x] **EXT-02**: 按 P1 抽取运营费率行（管理费、托管费等，可多行）
- [x] **EXT-03**: 抽取结果含字段级置信度与原文溯源（block/章节）
- [x] **EXT-04**: 枚举字段对照模板字典 sheet 校验（业绩基准、销售机构等）

### Excel Export

- [x] **XLS-01**: 基于 `templates/产品要素-2.xlsx` 模板填充「产品要素模板」sheet
- [x] **XLS-02**: 基于 `templates/产品运营费率导入模板-1.xlsx` 填充费率 sheet
- [x] **XLS-03**: 生成 xlsx 路径写入 DB；Phase 4 提供下载 API

### API & Frontend

- [x] **API-01**: `POST /upload` 接收 docx，创建任务（`pending`）
- [x] **API-02**: `GET /jobs/{id}` 返回解析/抽取状态与错误信息
- [x] **API-03**: 下载 Excel（产品要素、运营费率；后增锁定期、分级份额端点）

- [x] **UI-01**: 前端页面上传文件、展示任务状态、提供 Excel 下载

### DevEx

- [x] **DEV-01**: README 说明 venv、依赖、运行
- [x] **DEV-02**: pytest / CLI 验证解析→抽取→导出链路
